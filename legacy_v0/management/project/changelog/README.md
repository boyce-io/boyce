# Changelog & Archive

This directory contains active work logs and historical sprint artifacts.

## Structure

- **Active work:** `2025-12_WORK_LOG.md` — Current sprint work log
- **Archive:** `archive/` — Historical sprint artifacts, audits, and receipts

## Archive Organization

### Monthly Sprint Archives
- [`archive/2025-12/2025-12_sprint_archive.md`](archive/2025-12/2025-12_sprint_archive.md) — December 2025 sprint index

### Archive Categories
- **Audits:** [`archive/audits/`](archive/audits/) — Safety and compliance audit reports
- **Refactors:** [`archive/refactors/`](archive/refactors/) — Refactoring receipts and documentation
- **Notes:** [`archive/notes/`](archive/notes/) — Development notes and one-off documentation
- **Executive:** [`archive/EXECUTIVE_AUDIT_DEC_2024.md`](archive/EXECUTIVE_AUDIT_DEC_2024.md) — Executive-level audits
- **Planning:** [`archive/MARKDOWN_ARCHIVE_PLAN.md`](archive/MARKDOWN_ARCHIVE_PLAN.md) — Archive organization plan

## Lifecycle Rules

**Canonical sprint artifacts live only in `archive/YYYY-MM/`**

- Sprint evidence, reports, and receipts are archived in `project/changelog/archive/YYYY-MM/`
- No duplicate copies outside archive
- Files outside `archive/` should only be:
  - Index files (like this README)
  - Active work logs (like `2025-12_WORK_LOG.md`)
- When a sprint artifact is finalized, it must be moved to the appropriate `archive/YYYY-MM/` location
- Duplicate copies in `project/changelog/` root are not permitted
